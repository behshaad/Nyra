# Atomic Course Releases for Learner-Visible Structure

Course Structure, Skill, and Journey Presentation remain independent revision aggregates, but compatible reviewed revisions are promoted together as one atomic Course Release whenever Structure or Skill publication changes the learner-visible Practice Journey and requires matching presentation. This prevents Published hierarchy from referencing missing presentation or Published presentation from relying on incompatible structure without coupling ordinary content authoring to visual configuration.

A Course Structure Revision may publish independently only when it cannot change learner-visible hierarchy, such as adding an empty or Draft-only container. A Skill Revision may publish independently when the resulting learner-visible hierarchy remains valid against current Published Structure and Journey Presentation; publishing a Skill that makes a Level or Unit newly available requires the compatible Journey Presentation Revision in the Course Release. A Journey Presentation Revision may publish independently only when it validates against the current Published Course Structure. Release validation identifies required companion revisions, and learner rendering does not invent fallback coordinates to hide an incomplete release.
