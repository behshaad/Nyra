# Vercel deployment target

Nyra will use Vercel as the Deployment Target for the Next.js learner and admin application, and this repository should be linked to a new Vercel project named `nyra` rather than trying to restore the deleted `b1-mfis` project. Vercel production deployments should track the repository's `master` branch, with Codex work branches merged into `master` before production is expected to reflect them. This keeps production deployment aligned with the framework's default hosting path, while Nyra keeps product data, content, progress, and authentication boundaries in Nyra-owned or explicitly integrated services rather than treating the hosting platform as the product domain.
