# Vercel deployment target

Nyra will use Vercel as the Deployment Target for the Next.js learner and admin application, and this repository should be linked to a new Vercel project named `nyra` rather than trying to restore the deleted `b1-mfis` project. Vercel production deployments should track the repository's `master` branch, with Codex work branches merged into `master` before production is expected to reflect them. Production must use a managed PostgreSQL database exposed to Vercel through `DATABASE_URL`; the local `nyra_dev` database remains local-only. Because the repository currently has a Prisma schema and seed script but no migration history, the first production database bootstrap will use `prisma db push` followed by `npm run db:seed`, and future schema changes should introduce migrations before production use. This keeps production deployment aligned with the framework's default hosting path, while Nyra keeps product data, content, progress, and authentication boundaries in Nyra-owned or explicitly integrated services rather than treating the hosting platform as the product domain.
